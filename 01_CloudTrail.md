# CloudTrail（操作ログ）を理解

# CloudTrail とは？

「誰が・いつ・何をしたか」を記録する操作ログ。

【 記録される例 】

・S3バケットを作成した。

・バケットポリシーを変更した。

・EC2を起動・停止した。

【 記録されない例（重要） 】

・HTMLの中身。

・Webページを誰が見たか。

⇒ これは CloudTrailの仕事じゃない。

### CloudTrail ＝ 監視カメラ＋入退室記録簿

・誰が

・いつ

・どの扉を

・何の目的で開けたか

を全部記録する仕組み。


# 1. まずはイベント履歴を見てみる（いちばん簡単）

### 1-1. AWSコンソール ⇒ CloudTrail

### 1-2. 左メニュー ⇒ イベント履歴

### 1-3. 画面に一覧が出る

【 ここで見る列 】

・イベント名（PutObject / CreateBucket など）

・イベントソース（s3.amazonaws.com）

・ユーザー名

・イベント時刻

# 2. S3操作を絞り込んで見る

【 フィルター例 】

・イベントソース：s3.amazonaws.com

・イベント名：PutObject / PutBucketPolicy

【 確認ポイント 】

・自分が index.html をアップロードした記録があるか？

・バケットポリシーを設定した記録があるか？

 操作＝全部APIとして残ってるのが分かる。

 # 3. EC2操作も見てみる

【 フィルター例 】

・イベントソース：ec2.amazonaws.com

・イベント名：RunInstances / StopInstances

「EC2起動＝RunInstances」

AWSは全部APIで動いてると理解できる。

# 4. ログの中身を1件開いてみる（重要）

### 4-1. 任意のイベントをクリック

### 4-2. 詳細を開く

【 見るポイント 】

・eventName

・eventSource

・userIdentity

・requestParameters

「誰が」「どのサービスに」「何をしたか」が JSON で全部見える。

# 5. Trail（証跡）を作成する（ログ保存用）

・イベント履歴は 90日で消える。

・保存したい場合は Trail を作る。

### 5-1. CloudTrail → 証跡

### 5-2. 証跡の作成

### 5-3. 全般的な詳細（証跡名）

名前は何でもいい。意味が分かる名前で。

例 : week3-trail

### 5-4. 組織内のすべてのアカウントについて有効化

・今回はチェック外す。

【 チェックしない場合（学習・個人） 】

・今操作しているアカウントのみ。

・自分の操作ログだけ保存。

【 チェックした場合（実務向け） 】

・組織配下 すべてのアカウント

・すべての操作ログを1つのS3に集約

大企業・金融・医療で必須
